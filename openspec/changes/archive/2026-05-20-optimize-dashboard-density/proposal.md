## 为什么

当前新版界面已经完成主要信息架构，但视觉元素偏大、账号卡片占用高度过多，账号数量增加后首屏可见账号数不足。账号管理栏虽然功能方向正确，但布局不够整齐；底部 `数据每分钟自动同步` 提示与顶部 `最后同步` 重复，属于冗余信息。

## 变更内容

- 压缩整体界面密度：缩小标题、顶部卡片、账号卡片、头像、按钮、进度条和间距，让同屏可显示更多账号。
- 优化账号卡片布局：减少每行高度，保持剩余流量、reset 文案、状态徽标和切换按钮清晰可读。
- 重整账号管理栏：让标题区、展开按钮、操作按钮、选中计数和新增账号表单对齐一致，不再出现松散或割裂的面板结构。
- 彻底移除底部 `数据每分钟自动同步` 行；自动刷新行为保留，但只通过顶部 `最后同步` 表达。
- 保持小写 `couswee`、剩余流量 `% left` 语义、搜索、选择、删除、新增和切换功能不变。

## 能力范围

### 新增能力

无。

### 修改能力

- `account-dashboard-ui`：调整页面密度、账号管理栏布局，并移除底部自动同步提示。

## 影响范围

- 主要影响 `web/src/routes/+page.svelte` 与 `web/src/routes/global.css`。
- 不改变账号/用量/切换 API 语义。
- 验证需要覆盖前端构建、Go 测试、OpenSpec 校验，以及旧底部同步提示和旧大间距样式无残留。
