## 背景

当前界面按效果图重写后偏视觉展示型：卡片高、间距大、管理区分两层且留白较多。用户现在的目标是可长期承载更多账号，因此本轮优化应从“展示效果图”转向“密度更高但仍保持同一视觉风格”。

## 目标 / 非目标

**目标：**
- 增加账号列表在常见桌面视口中的可见数量。
- 账号管理栏改成更整齐的一体化工具条，减少不必要的分隔和大留白。
- 删除底部同步提示，避免与 `最后同步` 重复。
- 保留当前功能：搜索、选中、新增、删除、切换、剩余流量展示和自动刷新。

**非目标：**
- 不重做账号管理功能逻辑。
- 不引入虚拟列表或分页；本轮只做布局密度优化。
- 不改变后端 API 或数据语义。

## 决策

### 1. 优先通过 CSS 密度变量收缩布局
将大尺寸、宽间距和高卡片改为紧凑比例：缩小 shell 宽度边距、标题字号、summary 高度、账号卡片 padding、头像尺寸、按钮高度、进度条高度和行距。这样不改数据结构即可提升可见账号数。

备选方案：改成表格。拒绝原因：用户认可账号管理按钮方向和卡片式信息结构，问题是元素过大而不是卡片模型错误。

### 2. 账号管理栏改成一体化 toolbar
账号管理标题、折叠按钮、操作按钮和选中计数放入同一个工具条层级；展开新增表单时在工具条下方紧凑显示。避免之前 tab + body 两段式结构造成不整齐。

备选方案：保留两段式管理面板只调小尺寸。拒绝原因：两段式本身造成视觉割裂，用户明确指出实现不整齐。

### 3. 删除底部同步提示但保留轮询
移除 DOM 中的 `sync-footer` 和 CSS，继续保留 `setInterval(loadUsage, 60_000)` 与顶部 `最后同步`。同步状态集中在顶部表达，避免底部重复占用空间。

备选方案：缩小底部提示。拒绝原因：用户要求可以彻底清除。

## 风险 / 权衡

- [风险] 压缩过度导致可读性下降 → 缓解：只压缩容器、间距和按钮高度，保留关键百分比与按钮的视觉权重。
- [风险] 管理栏变紧凑后窄屏换行混乱 → 缓解：为 toolbar 和新增表单增加响应式折行规则。
- [风险] 删除底部提示后用户不知道会自动刷新 → 缓解：顶部 `最后同步` 仍持续更新，足以表达同步状态。
