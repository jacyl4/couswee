## 为什么

当前 couswee 仪表盘还偏向通用表格/管理后台，而新版效果图要求的是一个以“账号切换监测”为核心的暗色玻璃拟态单页界面：顶部概览、账号管理工具条、可选中的账号卡片、剩余流量条和一键切换。此次 change 以本轮新版效果图为最高 UI/功能准则，用来收束或替换旧 change 中与新版效果图冲突的界面描述。

## 变更内容

- 将当前表格优先的界面改为严格贴近新版效果图的单页布局：
  - 标题为 `couswee 账号切换监测`，副标题为 `个人多账号用量概览`。
  - 右上角提供账号搜索框，文案为 `搜索账号...`，并显示 `⌘K` 快捷提示。
  - 顶部显示 3 个概览卡：`可用账号`、`建议切换`、`最后同步`。
  - 概览区下方增加 `账号管理` 折叠式工具条，包含齿轮图标、展开/收起按钮、`新增账号`、`删除选中`，并在右侧显示 `已选择 N 项`。
  - 账号列表使用全宽卡片，每张卡片包含复选框、头像首字母、昵称、状态徽标、上次切换时间、`5h limit` / `Weekly limit` 剩余流量条、reset 时间和 `切换` 按钮。
  - 列表底部显示 `数据每分钟自动同步`。
- 新版效果图是此次 UI 的验收基准；视觉层级、间距、颜色、选中态、卡片状态和按钮样式都以它为准。
- 保留并强化“剩余流量”语义：界面展示 `93% left` 这类剩余百分比，不展示“已用量”。
- 增加账号选择功能：用户可以勾选账号；被选中的卡片需要出现边框高亮；`删除选中` 根据选中数量启用并展示选中数。
- 增加账号管理入口：`新增账号` 用于后续打开新增账号流程；`删除选中` 用于后续删除被选账号。若后端接口尚未完备，本 change 至少要把 UI 状态、按钮语义和任务拆分清楚。
- 旧 OpenSpec change 中与新版效果图冲突的 UI 表述可以删减、覆盖或不再采纳；账号注册、切换和用量采集能力只保留支撑新版界面所需的部分。
- 不新增 UI/chart 依赖；优先使用 SvelteKit 状态和 CSS 原生实现。

## 能力范围

### 新增能力
- `account-dashboard-ui`：定义新版 couswee 账号切换监测页，包括顶部概览、搜索、账号管理工具条、账号选择、批量删除入口、账号卡片、剩余流量展示、状态颜色和切换交互。

### 修改能力

无。当前项目还没有归档到 `openspec/specs/` 的正式能力；本能力作为下一轮 UI 实现的权威契约，覆盖旧未归档 change 中冲突的 UI 描述。

## 影响范围

- 前端文件：`web/src/routes/+page.svelte`、`web/src/routes/global.css`，必要时包括相邻 Svelte 组件/类型文件。
- 前端继续对接账号/用量接口：`GET /api/accounts`、`POST /api/accounts`、`DELETE /api/accounts`、`GET /api/codex/usage`、`POST /api/switch`。
- 新增/删除账号使用最小后端补充接口支撑新版效果图中的账号管理操作；本次核心仍是新版效果图界面。
- 验证应覆盖 Svelte 构建、类型检查、搜索/选择/阈值 helper、切换按钮行为，以及人工对照新版效果图的视觉检查。
- `build-couswee-gui`、`improve-codex-usage-monitor` 中旧的 UI 外观描述不再作为本次实现的优先依据。
